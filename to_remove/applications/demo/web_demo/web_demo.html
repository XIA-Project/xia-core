Backwards-compatible paths

  • Service Identifiers with fallback

    • Hello service (responding with "Hello World" message)
      http://www_s.hello.com.xia
      The name is resolved (via name service) to the following DAG format with fallback:
      http://dag/2,0/AD.serverAD=0:2,1/HID.serverHID=1:2/SID:hello_serviceSID=2:2

    • Web service for image (providing a list of CIDs that make up an image)
      http://www_s.xiaweb.com.xia/simple.html
      The name is resolved (via name service) to the following DAG format with fallback:
      http://dag/2,0/AD.serverAD=0:2,1/HID.serverHID=1:2/SID:web_servicesID=2:2//simple.html

    • Web service for XIA web page
      http://www_s.xiaweb.com.xia/xia/index.html
      The name is resolved (via name service) to the following DAG format with fallback:
      http://dag/2,0/AD.serverAD=0:2,1/HID.serverHID=1:2/SID:web_serviceSID=2:2//index.html

    • Stock ticker service (providing new info. every second)
      http://www_s.xiaweb.com.xia/stock.html
      The name is resolved (via name service) to the following DAG format with fallback:
      http://dag/2,0/AD.serverAD=0:2,1/HID.serverHID=1:2/SID:web_serviceSID=2:2//stock.html


  • Content Identifiers with fallback

    • Image content (a single CID for small image)
      http://www_c.airplane.com.xia
      The name is resolved (via name service) to the following DAG format with fallback:
      http://dag/2,0/AD.serverAD=0:2,1/HID.serverHID=1:2/CID:imageCID=2:2
